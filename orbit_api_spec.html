Section 01
API overview

The Orbit API is a RESTful HTTP API that provides programmatic access to all Orbit platform data and intelligence features. It is used internally by Orbit's web and mobile clients and is available to approved third-party integrations.

Base URL
https://api.orbitapp.io/v1
All endpoints are versioned. Breaking changes will be introduced in a new version with a 6-month deprecation window for v1.
Rate limits
Free tier: 100 requests / hour
Creator tier: 1,000 requests / hour
Pro tier: 5,000 requests / hour
Agency tier: 20,000 requests / hour
Rate limit headers returned on every response: X-RateLimit-Limit, X-RateLimit-Remaining, X-RateLimit-Reset

Request format

Headers

Authorization: Bearer {token} — required on all authenticated endpoints
Content-Type: application/json — required for POST/PUT/PATCH
X-Orbit-Version: 2026-03-01 — optional API version date pinning
Idempotency-Key: {uuid} — recommended for POST requests

Response format

All responses return JSON. Successful responses follow the envelope pattern:

{
  "success": true,
  "data": { ... },
  "meta": {
    "request_id": "req_abc123",
    "timestamp": "2026-03-18T10:00:00Z"
  }
}

Pagination

All list endpoints use cursor-based pagination for consistency and performance.

Paginated response envelope
{
  "success": true,
  "data": [ ... ],
  "pagination": {
    "cursor": "cur_eyJpZCI6MTIzfQ",
    "has_more": true,
    "total_count": 842
  }
}

Pass ?cursor= and ?limit= (max 100) as query params on subsequent requests.

Section 02
Authentication

Orbit uses JWT bearer tokens for API authentication. Tokens are obtained via the /auth endpoints and expire after 24 hours. Refresh tokens are valid for 30 days.

POST /auth/token Obtain access token Public

Request body

ParameterTypeDescription
emailrequiredstringUser email address
passwordrequiredstringUser password (min 12 chars)
mfa_codeoptionalstring6-digit TOTP code if MFA enabled
Response 200
{
  "access_token": "eyJhbGciOiJSUzI1NiJ9...",
  "refresh_token": "ref_abc123xyz",
  "expires_in": 86400,
  "token_type": "Bearer",
  "user": {
    "id": "usr_abc123",
    "email": "creator@example.com",
    "role": "creator",
    "tier": "pro"
  }
}
200 OK 400 Invalid credentials 401 MFA required 429 Rate limited
POST /auth/refresh Refresh access token Refresh token
ParameterTypeDescription
refresh_tokenrequiredstringValid refresh token from prior auth
POST /auth/oauth/{provider} OAuth login (Google, Apple) Public

Supported providers: google, apple. Pass OAuth id_token in request body. Returns same token envelope as /auth/token.

POST /auth/revoke Revoke token Bearer token

Revokes the current access token and optionally the refresh token. Pass "revoke_refresh": true to sign out all sessions.

POST /creators/me/connections/{platform}/sync Trigger manual data sync Bearer token

Triggers an immediate data sync for the specified platform. Rate limited to once per 15 minutes per platform. Returns a sync_job_id that can be polled via /jobs/{id}.

Note
Platform sync is performed automatically every 15 minutes. Manual sync is available for Pro and Agency tiers only.
DELETE /creators/me/connections/{platform} Disconnect a platform Bearer token

Revokes OAuth tokens for the specified platform and queues data deletion. Historical data is retained for 30 days before permanent deletion unless "delete_immediately": true is passed.

Section 06
AI insights

AI insight endpoints are powered by Orbit's intelligence engine. All responses are grounded in the creator's actual performance data. Responses may take 3–8 seconds — design for async/streaming where possible.

GET /insights/weekly Weekly strategy digest Bearer token

Returns this week's AI-generated strategic digest — prioritized action list, performance summary, and top opportunity. Generated every Monday at 6am UTC in the creator's timezone. Cached until next generation.

ParameterTypeDescription

Section 11
Webhooks

Orbit sends webhook events for real-time notifications. Configure webhook endpoints in your account settings or via the API. All payloads are signed with HMAC-SHA256 using your webhook secret.

Event types

EventTriggerTiers
escape_velocity.detectedContent crosses escape velocity thresholdAll
gravity_score.changedGravity Score changes by ±3 points or moreCreator+
trajectory.decliningOrbit Path detects 7-day declining momentumCreator+
dark_matter.briefing_readyWeekly Dark Matter briefing generatedCreator+
dark_matter.urgent_signalHigh-momentum signal detected (score >0.9)Pro+
sync.completedPlatform data sync completedAll
sync.failedPlatform data sync failed (auth error, API limit)All
report.readyAsync report generation completeAgency

Webhook payload structure

Escape velocity webhook payload
POST https://your-endpoint.com/orbit-webhook
Headers:
  X-Orbit-Signature: sha256=abc123...
  X-Orbit-Event: escape_velocity.detected
  X-Orbit-Delivery: del_xyz789

{
  "event": "escape_velocity.detected",
  "timestamp": "2026-03-12T09:22:14Z",
  "data": {
    "escape_velocity_id": "ev_abc123",
    "creator_id": "crt_abc123",
    "content_id": "cnt_xyz789",
    "platform": "instagram",
    "baseline_multiple": 6.2,
    "expires_at": "2026-03-14T07:14:00Z"
  }
}
Signature verification
Verify webhook authenticity by computing HMAC-SHA256(payload_body, webhook_secret) and comparing to the X-Orbit-Signature header. Reject any request where signatures do not match.
Section 12
Errors

All errors return a consistent JSON envelope with a machine-readable code and human-readable message.

Error response envelope
{
  "success": false,
  "error": {
    "code": "INSUFFICIENT_TIER",
    "message": "This endpoint requires a Pro or Agency subscription.",
    "docs_url": "https://docs.orbitapp.io/errors/INSUFFICIENT_TIER",
    "request_id": "req_abc123"
  }
}
