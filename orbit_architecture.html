Data ingestion flow
How a creator's post moves from Instagram to an AI insight
01 Platform API IG webhook fires or 15-min poll hits new post
→
02 Ingestion service Bull queue job · normalize to canonical schema
→
03 PostgreSQL write content + content_metrics rows inserted
→
04 AI engine job Baseline multiple calc · escape velocity check
→
05 Claude API call Generate analysis + replication playbook
→
06 Notification dispatch Push alert + WebSocket event to creator
Service specifications
Key technical details per service
Analytics service
Node.js 20 · Express · AWS ECS Fargate
Serves all dashboard API endpoints (/analytics/*, /gravity/*, /constellation/*)
Pre-computed aggregates cached in Redis with 15-min TTL
Read replica PostgreSQL connection for heavy queries
Target: <2s P95 dashboard load · 1,000 req/hr rate limit on Creator tier
AI engine service
Python 3.11 · FastAPI · AWS ECS Fargate
Gravity Score: nightly batch job at 2am UTC per creator
Escape velocity: event-driven trigger from ingestion service
Claude API calls: async with 8s timeout, exponential backoff retry
Custom ML: scikit-learn models for Orbit Path, Gravity Score components
Ingestion service
Node.js 20 · Bull · Redis · AWS ECS
Webhook handler for IG (instant) — polling fallback every 15 min
TikTok: polling every 15 min (no webhook support)
Queue priority: escape velocity detection jobs run at high priority
Metric snapshots captured at 1h, 6h, 24h, 72h, 7d, 30d post-publish
Trend service (Dark Matter)
Python 3.11 · Celery · Elasticsearch
Indexes audio track usage counts per niche every 6 hours
Momentum score recalculated daily via Celery beat scheduler
Weekly briefing generation: Celery task every Monday 5am UTC
Urgent signal alert: fires when momentum_score crosses 0.9 threshold
